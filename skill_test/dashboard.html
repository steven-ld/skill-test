Git-First Skill Benchmark

把 Skill 评测从命令行脚本，升级成可运营、可追踪、可复盘的平台产品。

平台同时支持 技术方案模式 与 Coding 模式。 在同一模型下，基线组只使用普通提示词，实验组引用项目 Skill 与按需勾选的 reference 文件； 最终交付以 Git 分支、提交和推送记录作为主凭证。

交付治理

实验结果默认围绕 Git 交付建立证据链。平台会记录每个实验组的 worktree 分支、 提交 hash、推送状态和交付文档路径，避免只看文本输出而无法复核。

双实验模式 同一任务可以在技术方案模式与 Coding 模式之间切换，解决“只能比代码、不能比方案”的缺口。
跨工具 Skill 目录 扫描结果会明确标注 `claude / cursor / codex / gemini` 等来源，避免前端误把不同工具的 Skill 混为一谈。
按需引用 reference 发现 Skill 后不会自动全量注入，需要人工勾选要引用的参考文件，便于做严格 A/B 对照。
